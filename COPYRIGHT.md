## Attribution Guidelines

As specified in the [LICENSE](file:///F:/opened-git/LICENSE), you are allowed to reuse, modify, and distribute this codebase or parts of it, provided you adhere to the following attribution guidelines:

### 1. Source Code Files
If you copy or modify any source code files (Rust files in `src-tauri/`, TypeScript/React files in `src/`, or configuration files), you must preserve the copyright headers at the top of the files. If no header exists, you must add one indicating the original author.

### 2. Forking or Reusing the Entire Project
If you fork this repository or build a project derived from it, you must:
- Keep the `LICENSE` and `COPYRIGHT.md` files intact in the root directory.
- Provide a clear link back to the original repository: [VesperAkshay/tyegit](https://github.com/VesperAkshay/tyegit).
- Add a visible attribution notice in your project's `README.md` or landing page:
  > *This project is based on/derived from the Git Desktop Client originally developed by VesperAkshay.*

### 3. Application User Interface (About / Credits Screen)
If your derived application features a User Interface (UI), you must include a "Credits" or "About" section that mentions the original author. For example:
- **"Git Desktop Client core engine & design developed by VesperAkshay."**
